## 추가 · Added

- `wiki_block_version` 인식 업그레이드 리포트 — `migrate`/`doctor`가 각 문서의 기록된 블록버전과 설치된 CLI 사이의 계약 갭을 보여준다(stamp 값의 단일 소스는 `CURRENT_WIKI_BLOCK_VERSION`). · a `wiki_block_version`-aware upgrade report in `migrate`/`doctor` (the stamped value now has a single source, `CURRENT_WIKI_BLOCK_VERSION`).
- `migrate --apply` 해금 — 승인된 미리보기 우선·`verified` 보존 범위(GATE_REVIEW Gate 8). `fix` 엔진 + `wiki_block_version` 업그레이드를 재사용하고, 문서가 부합해지면 블록버전을 stamp한다. `status`나 `verified` 내용은 불변, ahead 문서는 다운그레이드하지 않는다. · unblocked `migrate --apply` under an accepted, preview-first, `verified`-preserving scope (Gate 8), reusing the `fix` engine plus a `wiki_block_version` upgrade.
- `llm-wiki drift` 명령 — `verified` 문서의 `evidence.stale` 드리프트를 보고하고, `--downgrade`로 드리프트된 문서를 `needs_review`로 내린다(`status`·`last_updated`만; GATE_REVIEW Gate 9). · a new `drift` command that reports drift and, with `--downgrade`, moves drifted `verified` docs to `needs_review` (Gate 9).
