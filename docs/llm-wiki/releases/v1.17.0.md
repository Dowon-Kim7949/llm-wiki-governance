## 요약 · Summary

- **reverse-impact 게이트(Gate 23).** 날짜 기반 drift가 놓치는 "코드와 그 `verified` 문서가 **다른 PR**에서 바뀌는" 경우를 잡는 읽기 전용 `impact` 명령을 추가한다. 부가적·opt-in이라 `llm-wiki` 명령 표면은 하위호환이고 `--format json`·프로그래매틱 API·frontmatter 계약·zero-dep 불변. · Reverse-impact gate (Gate 23). Adds a read-only `impact` command that catches the case date-based drift misses — code and its `verified` doc changing in **separate** PRs. Additive/opt-in; the command surface, `--format json`, the programmatic API, the frontmatter contract, and zero-dep are unchanged.

## 추가 · Added

- **`llm-wiki impact` — diff 기준 reverse-impact 체크(읽기 전용).** 모든 `verified` 문서의 로컬 `source_files`/`evidence`에서 역색인을 만들어, 참조한 소스가 현재 변경집합에 들어 있는데 **문서 자신은 같은 diff에서 바뀌지 않은** `verified` 문서를 flag한다. 날짜 기준 `evidence.stale`(drift)의 **pre-merge·diff 기준 보완**이다(대체 아님). · A read-only, diff-anchored reverse-impact check. Builds a reverse index from every `verified` doc's local `source_files`/`evidence`, then flags a `verified` doc whose referenced source is in the current change set **while the doc itself is unchanged in the same diff**. The pre-merge, diff-anchored complement to the date-anchored `evidence.stale` (drift), not a replacement.
- **변경집합 기준 · Change-set baseline.** 기본은 **working tree**, PR/CI용은 `--since <ref>`(`git diff --name-only <ref>`) — `validate --changed`가 쓰는 `changedFiles` 프리미티브 재사용. **빈 변경집합은 no-op**(result `pass`). · Working tree by default, `--since <ref>` for a PR/CI base; reuses the `changedFiles` primitive. An empty change set is a no-op.
- **finding · Findings.** 신규 `impact.source_changed`(신규 **toggleable** `impact` 카테고리, 기본 **warning**), git이 없으면 `impact.unavailable`(error). · New `impact.source_changed` (new toggleable `impact` category, default warning); `impact.unavailable` (error) when git is unavailable.
- **`--strict`(CI).** impact findings를 실패 error로 승격해 governed 코드를 바꾸면서 `verified` 문서를 안 고친 PR을 CI에서 fail시킨다. severity는 config `rules` 맵으로도 조정 가능(`evidence.stale`은 그대로 config로 승격 가능). · `--strict` escalates impact findings to a failing error for CI; severity is also adjustable via the config `rules` map.
