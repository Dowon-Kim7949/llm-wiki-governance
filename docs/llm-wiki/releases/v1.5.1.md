# 릴리스 노트 v1.5.1 · Release Notes v1.5.1

## 요약 · Summary

- 1.5 신규 프로그래매틱 API를 소비 프로젝트에서 스모크 테스트하다 발견한 API/출력 결함 4건 수정. 모두 additive/refinement라 안정 계약(CLI·JSON·frontmatter)은 그대로다. · Four API/output fixes found while smoke-testing the new 1.5 API from a consuming project. All additive/refinements; the stable CLI/JSON/frontmatter contracts are unchanged.

## 수정 · Fixed

- 명령 결과 객체가 최상단 `schemaVersion`(= `SCHEMA_VERSION`)을 항상 담는다. `.text`는 어떤 경우에도 텍스트 리포트이며 `--format`은 CLI/`run()`·`--out`에만 영향(문서 명시). · result objects now carry `schemaVersion`; `.text` is always the text report and `--format` only affects CLI/`run()`/`--out`.
- `normalizeOptions`가 `parseArgs(argv)` 결과를 그대로 수용(중첩 `.options` 인식) → 조용한 기본값 폴백 제거. · `normalizeOptions` accepts a `parseArgs` result directly, removing the silent default fallback.
- `run(argv)`가 숫자 exit code(0/1/2/3)를 반환(+`process.exitCode`도 설정). · `run(argv)` now returns the numeric exit code.
- `--format html` 대시보드 Document Index 링크를 `--out` 위치 기준 상대경로로 계산 → 하위 폴더 대시보드의 깨진 링크 해결. · dashboard Document Index links are now relative to the `--out` location, fixing broken links in subfolder dashboards.

## 참고 · Notes

- 발견 경위: 소비 프로젝트(road-monitor)의 in-process 스모크 테스트. graph DOT 결정성 등 "정상"으로 확인된 항목은 변경하지 않았다. · Found via a consuming project's in-process smoke test; items confirmed working (e.g. deterministic graph DOT) were left unchanged.
