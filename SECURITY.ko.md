# 보안 정책

## 지원 버전

`@dowonk-7949/llm-wiki-standard`는 롤링 릴리스 모델을 따르며, 보안 수정은 npm의
**최신 배포 버전**에 반영됩니다. 신고 전에 최신 버전으로 업그레이드하고,
프로젝트에서는 최근 버전으로 고정해 사용하세요.

| 버전 | 지원 |
|---|---|

## 취약점 신고

**보안 취약점은 공개 GitHub 이슈로 열지 마세요.**

아래 비공개 채널 중 하나를 이용해 주세요:

1. **GitHub 비공개 취약점 신고 (권장)** — 저장소의 **Security** 탭 →
   **Report a vulnerability**. 신고가 비공개로 유지되고, 수정과 보안
   권고(advisory)를 함께 조율할 수 있습니다.
2. **이메일** — ungong0328@gmail.com.

가능한 범위에서 다음을 포함해 주세요:

- 문제 설명과 영향.
- 재현 절차(최소한의 명령 시퀀스나 저장소 상태).
- 영향 받는 버전과 환경(OS, Node.js 버전).
- 제안하는 해결 방안.

## 처리 기대치

이 프로젝트는 소규모 팀이 최선 노력(best-effort) 기반으로 관리합니다:

- **접수 확인:** 영업일 기준 약 5일 이내.
- **평가 및 수정:** 진행 상황을 공유하며, 문제가 확인되면 다음 릴리스에서 수정을
  목표로 합니다.
- **공개:** 조율된 공개(coordinated disclosure)를 선호합니다. 수정이 배포되면
  원하는 신고자를 GitHub 보안 권고에 크레딧으로 표기합니다.

## 범위 및 위협 모델

신고 범위를 잡는 데 도움이 되는 사실:

- 이 CLI는 **런타임 서드파티 의존성이 없습니다**(Node.js 내장 모듈만 사용) —
  의존성 공격면이 최소화됩니다.
- **로컬 파일시스템**에서 동작합니다 — 프로젝트 파일을 읽고, 명시적
  `--write`/`--apply` 플래그가 있을 때에만 wiki/adapter 파일을 씁니다. 프로젝트
  내용을 외부로 전송하지 않습니다.
- 리포트에서 의심되는 비밀값을 표시·마스킹하는 **민감정보 스캔** 기능이 있습니다.

특히 관심 있는 신고: 의도 범위를 벗어난 쓰기, 경로 탐색(path traversal), 조작된
프로젝트 파일을 통한 코드 실행, 리포트/로그로의 민감정보 유출.

프로젝트와 사용자의 안전을 지키는 데 도움 주셔서 감사합니다.
